## Prerequisites

- Rust toolchain (stable, 2021 edition)
- [Trunk](https://trunkrs.dev) for building and serving WASM apps
- The `wasm32-unknown-unknown` target

```bash
rustup target add wasm32-unknown-unknown
cargo install trunk
```

## Add the dependency

Add `rs-grid-leptos` to your `Cargo.toml`. While the crate is not yet on
crates.io, reference it by path:

```toml title="Cargo.toml"
[dependencies]
rs-grid-leptos = { path = "../rs-grid-leptos" }
```

## Basic usage

Import the component and mount it inside a Leptos view:

```rust title="src/main.rs"
use leptos::prelude::*;
use rs_grid_core::{column::ColumnDef, model::GridModel};
use rs_grid_leptos::GridCanvas;

#[component]
pub fn App() -> impl IntoView {
    let columns = vec![
        ColumnDef::new("name", "Name", 200.0),
        ColumnDef::new("email", "Email", 260.0),
    ];
    let data = vec![
        vec!["Alice".into(), "alice@example.com".into()],
        vec!["Bob".into(), "bob@example.com".into()],
    ];
    let model = GridModel::new(columns, data, 32.0, 40.0);

    view! {
        <main style="width: 100vw; height: 100vh;">
            <GridCanvas model=model width="100%" height="100%" />
        </main>
    }
}

fn main() {
    mount_to_body(App);
}
```

<Tip>
  For large or infinite datasets, build the model from a `FnDataSource`
  (`GridModel::with_data_source`) instead of an in-memory `Vec` — this is the
  path behind the millions-to-quadrillions row figures.
</Tip>

## Run locally

<Steps>
  <Step title="Clone an example repo">
    ```bash
    git clone https://github.com/ruxelion/rs-grid-example-leptos
    cd rs-grid-example-leptos
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    trunk serve
    ```
  </Step>
  <Step title="Open in your browser">
    Navigate to `http://localhost:9080`. You should see the demo grid
    rendering on a `<canvas>`.
  </Step>
</Steps>

## Build for production

```bash
cd rs-grid-example-leptos
trunk build --release
```

The output goes into `dist/`. Serve it with any static file server or use the
provided [Docker image](/deployment/docker).

## Workspace commands

| Command | Description |
|---------|-------------|
| `cargo check --workspace` | Quick type-check of all crates |
| `cargo test --workspace` | Run all unit tests |
| `cargo fmt --all` | Format the entire workspace |
| `cargo clippy --workspace -- -D warnings` | Lint with warnings as errors |
| `trunk serve` (in example dir) | Dev server with hot reload |

<Tip>
  `rs-grid-core` has no WASM dependency — its unit tests run with plain
  `cargo test`, no browser needed.
</Tip>
