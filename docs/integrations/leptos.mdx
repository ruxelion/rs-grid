`rs-grid-leptos` provides a `<GridCanvas>` component for Leptos CSR
applications. It wraps the WASM runtime, canvas lifecycle, event handling, and
theming in a single component.

## Component API

`<GridCanvas>` takes a `GridModel` plus CSS sizing. Build the model from
columns + data (or a data source) and hand it to the component:

```rust
use leptos::prelude::*;
use rs_grid_core::{column::ColumnDef, model::GridModel};
use rs_grid_leptos::GridCanvas;

let columns = vec![
    ColumnDef::new("name", "Name", 200.0),
    ColumnDef::new("email", "Email", 260.0),
];
let data = vec![
    vec!["Alice".into(), "alice@example.com".into()],
    vec!["Bob".into(), "bob@example.com".into()],
];
let model = GridModel::new(columns, data, 32.0, 40.0);

view! {
    <GridCanvas model=model width="100%" height="600px" />
}
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `model` | `GridModel` | required | Columns + data (or data source). Moved into the component on first render. |
| `width` | `String` | `"100%"` | CSS width, e.g. `"100%"` or `"800px"`. |
| `height` | `String` | `"600px"` | CSS height, e.g. `"600px"`. |
| `theme` | `Signal<Theme>` | optional | Reactive theme â€” changes apply in place without remounting. |
| `locale` | `Signal<Locale>` | optional | Reactive locale â€” changes apply in place without remounting. |
| `on_mount` | `Box<dyn FnOnce(WebGridCanvas)>` | optional | Receives a handle after mount; use for `set_on_change`, `import_patches`, `export_patches`. |
| `on_validation_error` | `Box<dyn Fn(u64, String, String)>` | optional | Fired when a column validator rejects an edit: `(row, col_key, message)`. |
| `on_cell_button_click` | `Box<dyn Fn(u64, String, String)>` | optional | Fired when a cell button is clicked: `(row, col_key, button_id)`. |

<Tip>
  `GridModel` is consumed (moved) on first render â€” it is intentionally not
  `Clone`, because `FnDataSource` closures are not clonable. For large or
  infinite datasets, build it with `GridModel::with_data_source` instead of an
  in-memory `Vec`.
</Tip>

## Theming

The component reads its palette from `--rs-grid-*` CSS custom properties at
mount time via `rs-grid-web::theme_from_css_vars`. Define them on `:root`:

```css
:root {
  --rs-grid-bg:               #ffffff;
  --rs-grid-header-bg:        #f8f9fb;
  --rs-grid-header-text:      #181d1f;
  --rs-grid-cell-text:        #181d1f;
  --rs-grid-grid-line:        #e2e8f0;
  --rs-grid-header-border:    #babfc7;
  --rs-grid-selection-fill:   rgba(33, 150, 243, 0.20);
  --rs-grid-selection-border: rgba(33, 150, 243, 0.85);
}
```

See the [CSS Variables Reference](/theming/css-variables) for the complete
list, and `examples/example-common/themes/light.css` / `dark.css` for full
ready-to-use themes.

## Events

The Leptos component attaches pointer and wheel listeners to the canvas:

| Browser event | GridCommand |
|---------------|-------------|
| `pointerdown` | `SelectCell` / `SelectRow` / `SelectColumn` |
| `pointerdown` + Shift | `ExtendSelection` |
| `wheel` | `ScrollTo` |
| `ResizeObserver` | `Resize` |

Events are translated to `GridCommand` values and applied on the next animation
frame. You do not need to manage the event loop manually.

## Full example

```rust title="src/main.rs"
use leptos::prelude::*;
use rs_grid_core::{column::ColumnDef, model::GridModel};
use rs_grid_leptos::GridCanvas;

#[component]
pub fn App() -> impl IntoView {
    let columns = vec![
        ColumnDef::new("product", "Product", 220.0),
        ColumnDef::new("price", "Price", 120.0),
    ];
    let data = vec![
        vec!["Keyboard".into(), "49.99".into()],
        vec!["Monitor".into(), "199.00".into()],
    ];
    let model = GridModel::new(columns, data, 32.0, 40.0);

    view! {
        <main style="width: 100vw; height: 100vh;">
            <GridCanvas model=model width="100%" height="100%" />
        </main>
    }
}

fn main() {
    mount_to_body(App);
}
```
